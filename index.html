这里是您的专属Docker镜像仓库，致力于为您提供高效、便捷的镜像服务。

为了让您能够顺利地拉取和使用本仓库中的镜像，请参考以下操作指南：

  • 方法一：直接使用 `docker pull` 命令拉取镜像
    如果您的Docker守护进程配置允许非HTTPS注册表（不推荐用于生产环境），您可以直接使用`docker pull`命令。 
    docker pull liuasd-dcor.edgeone.app/your-image-name:tag

    注意： 由于 `liuasd-dcor.edgeone.app` 使用的是非标准端口或自签名证书（如果EdgeOne没有提供公共CA信任的证书），Docker客户端默认会拒绝连接。因此，强烈建议您使用方法二来配置Docker守护进程。

  • 方法二：配置Docker守护进程信任此不安全注册表（推荐）
    为了让Docker守护进程能够从 `liuasd-dcor.edgeone.app` 拉取镜像，您需要将其添加为“不安全注册表”（insecure-registries）。这通常适用于私有或内部网络中的非HTTPS注册表。

    操作步骤：

    1. 编辑或创建 `daemon.json` 文件：
      在您的Docker主机上，找到或创建Docker守护进程的配置文件 `daemon.json`。
      通常位于：
      • Linux: `/etc/docker/daemon.json`
      • Windows (WSL2/Docker Desktop): 在Docker Desktop设置中配置，或在WSL2环境中找到相应路径。
      • macOS (Docker Desktop): 在Docker Desktop设置中配置。

      使用您喜欢的文本编辑器打开此文件，例如：

      sudo vim /etc/docker/daemon.json
    2. 添加 `insecure-registries` 配置：
      如果文件为空或不存在，请添加以下内容： 
      {
  "insecure-registries": [
    "liuasd-dcor.edgeone.app"
  ]
}
      如果文件中已有其他配置，请确保将其添加到现有JSON对象的顶部或底部，并用逗号分隔（如果不是最后一个条目）： 
      {
  "other-config": "value",
  "insecure-registries": [
    "liuasd-dcor.edgeone.app"
  ]
}
    3. 重启Docker守护进程：
      保存 `daemon.json` 文件后，您需要重启Docker守护进程以使更改生效。
      • Linux (Systemd): 
        sudo systemctl daemon-reload
sudo systemctl restart docker
      • Docker Desktop (Windows/macOS):

        在Docker Desktop应用中，点击“Restart Docker”或“Apply & Restart”按钮。

    4. 验证配置并拉取镜像：
      重启后，您现在应该能够成功从 `liuasd-dcor.edgeone.app` 拉取镜像了： 
      docker pull liuasd-dcor.edgeone.app/your-image-name:tag
  • 镜像命名规范：
    请确保您拉取的镜像名称前缀为 `liuasd-dcor.edgeone.app/`。例如，如果您要拉取名为 `my-app` 且标签为 `latest` 的镜像，完整的拉取命令将是：
    docker pull liuasd-dcor.edgeone.app/my-app:latest
  • 遇到问题？
    如果在配置或拉取过程中遇到任何问题，请检查您的网络连接、`daemon.json` 文件语法是否正确，并确保Docker守护进程已成功重启。如果问题依然存在，请联系仓库管理员获取支持。
